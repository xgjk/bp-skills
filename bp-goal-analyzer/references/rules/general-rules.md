# 通用规则

> 本文件为强制约束，贯穿单目标分析全流程。

## ID 处理

- 所有 ID 参数保持字符串原样传递，**严禁 parseInt 或 Number 转换**

## 数据真实性

- **严禁伪造** R 编号、汇报链接、reportId 或任何数据
- 证据引用格式：`[R编号](huibao://view?id={reportId})`，不附带书名号标题
- R 和 RP 编号使用不同前缀，严禁混用

## AI 产出约束

- AI 产出结构化 JSON 片段，**禁止在 JSON 文本字段中使用 Markdown 格式化**
- 禁止一次性产出完整 `goal_complete.json`，必须按 3 个片段分步产出
- 文本字段使用中文，正式商务风格，句式自然（主谓宾完整）
- 禁止空值直出（如"无数据"），转为有引导意义的自然语句
- 禁止技术字段泄漏（reportId、taskId 等 API 字段名不得出现在文本中）

## 上下文隔离

- 禁止读取其他目标的中间文件
- 每次只处理一个目标（goalId），不依赖其他目标的数据

## 容错与重试

- 数据采集失败：检查脚本输出的 `errors` 数组，核心数据已获取则继续，缺失则重试一次
- 校验失败：AI 修正对应片段 → 重新拼装 → 重试校验，最多 2 次
- 任何单步最多重试 2 次
- BP 数据获取失败时，提示用户检查 `BP_OPEN_API_APP_KEY` 配置
