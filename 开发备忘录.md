# 开发备忘录

> 目标：记录开发过程中遇到的问题与解决方案，后续遇到类似问题优先在此查找。

## 2026-04-08

### 1) 写入 Skill 需要“写前确认单/写后复核”，但写 Skill 原本只实现写接口

**问题**：`cms-bp-manager-write` 需要在执行 `updateTask` 等写操作前生成差异确认单，并在写后复核落地结果；但原实现只封装写接口，没有读取详情的能力。

**解决**：在写入侧的 `bp_client.py` 增加最小只读方法（`GetGoalDetail` / `GetKeyResultDetail` / `GetActionDetail`），仅用于写前/写后验证，不替代 `cms-bp-manager` 的查看/审计入口；CLI 的 `update-task` 以这些只读方法实现差异确认单与复核。

### 2) 对齐脚本从“占位禁止编造”迁移到真实 OpenAPI

**问题**：历史占位脚本 `set_alignment.py` 仅提示“未接入禁止编造”，无法支持后端已上架的对齐接口。

**解决**：按 OpenAPI（二、2.26）接入 `POST /bp/task/v2/alignTask`，并用 `upwardTaskIdList=[]` 表达解除对齐。
