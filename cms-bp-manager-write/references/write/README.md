## write 模块说明（写入更新）

### 适用场景

用于 BP 的写入更新与维护动作（新增/修改/删除/对齐承接维护等）。本仓库当前落地以“已明确 Open API 支持的写操作”为准，其他写操作必须显式提示“需前端人工操作”，禁止编造接口。

### 写入动作的安全约束

1. 写入前必须先输出“变更影响说明 + 风险点 + 审计结论”
2. 写入动作必须二次确认后执行
3. 写入后必须再复核（可调用审计入口）

### 当前已接入的写操作（以脚本实现为准）

- 新增关键成果（`addKeyResult`）
- 新增关键举措（`addAction`）
- 发送延期提醒（`sendDelayReport`）
