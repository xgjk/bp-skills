## reporting 模块说明

### 适用场景

本模块用于生成 BP 报告链路中的“填写规范”，包括：月报、季报、半年报、年报。

### 输入最小集

- `periodId`：BP 周期 ID
- `orgName` 或 `groupId`：组织节点名称或分组 ID（二选一）
- `templateTypes`：报告类型（`月报/季报/半年报/年报/四套`）
- `outputDir`：输出目录

### 输出

- 在输出目录生成对应“填写规范” Markdown 文件（并可能生成审查未通过的 DRAFT 与审查 JSON）
- 如输入不足以唯一定位周期或组织节点，应停止生成并提示用户补充

### 鉴权约束

- 所有鉴权统一依赖 `cms-auth-skills`
- 业务脚本仅接收运行时注入的 `appKey`（不硬编码、不回显）
